<objective>
Lower shared-memory `ttg.extern_call` operands directly as `ptr addrspace(3)`, bypassing the distributed `alloca+store+ptr` path, while preserving subview offsets via `getShmemAffineBase`. Distributed operands keep their existing path in mixed argument lists, matching callee signature order.

Purpose: The LLVM IR call instruction must show `ptr addrspace(3)` for shared-memory arguments and `ptr addrspace(0)` for tensor arguments, with correct GEP-computed subview offsets applied to shared bases. The lowering is spec-JSON-driven via the `ttg.extern_call_arg_spaces` module attribute.
Output: Modified `ExternCallOpToLLVM.cpp` (new `getArgMemorySpaces` helper, per-operand shared branch in matchAndRewrite) and new lit test `extern-call-shared-args.mlir`.
</objective>
