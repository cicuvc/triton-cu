     This method constructs a `SharedLinearLayout<OffsetBases{RANK,N_OFFSET}, BlockBases{RANK,N_BLOCK}, Align>` clang type.
     ```
     Steps:
     1. Determine RANK from info.OffsetBasis: if info.OffsetBasis is non-empty and RANK is set, use info.RANK.
        Otherwise, auto-detect: RANK = info.OffsetBasis.size() / N_OFFSET_BASES (where N_OFFSET_BASES = number of rows).
        For Phase 4, we assume row count = 1 for simplicity; or compute from the flattened data.
        Actually: simply use info.RANK directly (caller must set it).
     2. Determine N_OFFSET_BASES and N_BLOCK_BASES:
        - N_OFFSET = info.OffsetBasis.size() / info.RANK (if info.OffsetBasis non-empty; else 0)
        - N_BLOCK = info.BlockBasis.size() / info.RANK (if non-empty; else 0)
     3. Build OffsetBases<RANK,N_OFFSET> carrier:
        a. Look up OffsetBases template: getTemplateDecl(Ctx, "OffsetBases")
        b. Create TemplateArgumentList with {mkIntegralArgUint32(RANK), mkIntegralArgUint32(N_OFFSET)}
        c. findSpecialization → if not found, ClassTemplateSpecializationDecl::Create + AddSpecialization
        d. Skip InstantiateClassTemplateSpecialization for the carrier (no Sema semantic checks needed)
        e. Get QualType for the OffsetBases specialization
     4. Build OffsetBases NTTP value (same pattern as BuildBasisGroup lines 239-282):
        - For each row i in [0, N_OFFSET):
          * Extract RANK values from info.OffsetBasis[i*RANK ... (i+1)*RANK-1]
          * Build IntTuple<RANK> via the existing BuildIntTuple helper
          * Wrap in APValue with IntTuple struct + array fields
        - Create the final APValue array of IntTuple values
        - Create TemplateParamObjectDecl
        - Return {TemplateArgument(TPOD, canonical type), carrier specialization}
     5. Build BlockBases<RANK,N_BLOCK> carrier identically (using info.BlockBasis)
     6. Build SharedLinearLayout specialization:
        a. Look up SharedLinearLayoutTemplateType (set in constructor)
        b. Template args: {OffsetBases NTTP, BlockBases NTTP, mkIntegralArgUint32(info.Alignment)}
        c. findSpecialization → Create if not found
        d. Skip InstantiateClassTemplateSpecialization (Phase 4 does not need the definition instantiated for type building)
        e. Return Ctx.getTemplateSpecializationType(Struct, TemplateName(SharedLinearLayoutTemplateType), args...)
     ```

     Simplified approach for Phase 4 (when OffsetBases has only 1 row):
     - Build OffsetBases with N_OFFSET=1 (or however many rows the input data indicates)
     - The NTTP array uses the BuildBasisGroup APValue pattern (lines 261-282) but adapted for a top-level template:
       the TemplateParamObjectDecl is created against the OffsetBases<RK,N> specialization's canonical type
     - If info.OffsetBasis is empty: create OffsetBases with N_OFFSET=0 (empty array — the template supports zero bases)

     clang::QualType CUDACompiler::BuildSharedTensor(const SharedTensorParameter &Param) {
       clang::QualType Result;
       TaskQueue.emplace([&](TensorTypeHelpers &helper, CustomAstConsumer &) {
         // 1. Build Shape type from Param.Shape (reuse existing helper.Builder.buildShape)
         auto Shape = helper.Builder.buildShape(Param.Shape);

         // 2. Build SharedLinearLayout type from Param.Layout
         auto Layout = helper.Builder.BuildSharedLinearLayout(Param.Layout);

         // 3. Build SharedTensor type
         Result = helper.Builder.BuildSharedTensor(

             Shape.type, Layout);
       });
       InvocationContext->SwitchTo(*CompileExecutionContext);
       return Result;
     }
