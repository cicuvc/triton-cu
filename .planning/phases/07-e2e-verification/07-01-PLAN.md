**Function 1: `shared_accumulate`** (per D-26, SHTEST-01 mixed-args)

```cpp
template<typename T, uint32_t N, typename SharedTLayout, typename TLayout>
__device__ void shared_accumulate(
    SharedTensor<T, Shape<N>, SharedTLayout>& shm,
    const Tensor<T, Shape<N>, TLayout>& val)
{
    #pragma unroll TLayout::REG_SIZE
    for (uint32_t i = 0; i < TLayout::REG_SIZE; i++)
        shm(i) += val.data[i];
}
```

**Key design notes:**
- FOUR template parameters (not three): `SharedTLayout` for SharedTensor's byte-offset evaluation, `TLayout` for Tensor's `REG_SIZE` — RESEARCH.md confirms `SharedLinearLayout` has no `REG_SIZE` member; must use separate types
- `#pragma unroll TLayout::REG_SIZE` — matches existing `elementwise_add` (line 232) and `intra_warp_add_sibling` (line 250) conventions
- `shm(i)` calls `SharedTensor::operator()(auto...)` (line 185) — single-index overload handles 1D shapes correctly
- `val.data[i]` uses `Tensor::data` array (line 87) — register-level access, not operator()

**Function 2: `write_swizzled_2d`** (SHTEST-02 swizzle test)

```cpp
template<typename T, typename TLayout>
__device__ void write_swizzled_2d(SharedTensor<T, Shape<32, 16>, TLayout>& shm) {
    for (int i = 0; i < 32; i++)
        for (int j = 0; j < 16; j++)
            shm(i, j) = static_cast<T>(i * 16 + j);
}
```

**Key design notes:**
- Same template signature as `process_shared_2d` (line 220): `T` + `TLayout` only — no N (shape is fixed at `Shape<32,16>`)
- Double loop `i`/`j` matches `process_shared_2d` iteration pattern (lines 221-224)
- `shm(i, j) = static_cast<T>(i * 16 + j)` — writes identity value (logical index as float), later read back via identity layout in Python test
- `static_cast<T>` ensures type compatibility when T is not int

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Tampering | `shared_accumulate` loop bound | medium | mitigate | Loop bound is `TLayout::REG_SIZE` (from the distributed tensor), not `N` (from shape). For non-identity layouts where `REG_SIZE < N`, this prevents out-of-bounds shared memory writes. For identity layouts, `REG_SIZE == N` and all elements are covered. |
| T-07-02 | Information Disclosure | `write_swizzled_2d` shared memory | low | accept | Function writes deterministic identity values (`i*16+j`) to shared memory — no user data is written to shared memory by this function. The data is seed data for the round-trip test only. |
| T-07-03 | Tampering | `SharedTensor::operator()` byte-offset computation | low | accept | Byte-offset arithmetic uses `uint32_t` for `flatIndex` and `byteOffset`. For Shape<32,16> (512 elements × 4 bytes = 2048 bytes), no overflow risk. For Shape<256> (1D, 1024 bytes), no overflow risk. These are compile-time constant shapes. |
