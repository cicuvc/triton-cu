## Part B: Implement LookupFunctionWithPlaceholderFallback in clang_compiler.cc

Insert the method body between `CUDACompiler::LookupFunction` (ends at line 790) and `CUDACompiler::InstantiationFunction` (starts at line 792). The implementation:

1. Look up the FunctionTemplateDecl by Name using Ctx.getTranslationUnitDecl()->lookup(&Ctx.Idents.get(Name)). Filter to FunctionTemplateDecls only.

2. For each matching FunctionTemplateDecl FTD:
   a. Build PlaceholderLayout arg types: iterate over ParamTypes, for each TensorParameter variant, create a local copy with N_WARPS=0, LayoutShape from the original, and all bases empty (default-constructed). Leave ScalarType params unchanged. Call BuildTensor on each modified TensorParameter — this hits the PlaceholderLayout branch at lines 716-737 because N_WARPS==0 and bases are empty.
   
   b. Extract element type for explicit template args: from the first arg's TensorParameter, call getQualTypeFromScalarType(helper.Builder.Ctx, firstParam.Type) to get the clang QualType for the scalar (e.g., FloatTy). This will serve as the explicit template argument for a single type template parameter.

   c. Build a TemplateArgumentListInfo (TALI) with one argument: clang::TemplateArgument(Ctx.getCanonicalType(elementQualType)). Use clang::TemplateArgumentLoc(TArg, clang::TemplateArgumentLocInfo()) to add to TALI.

   d. Build OpaqueValueExpr callArgs from the PlaceholderLayout arg types (same pattern as LookupFunction at lines 489-493).

   e. Call SemaRef.DeduceTemplateArguments(FTD, &TALI, placeholderCallArgs, cand, DI, false, false, false, QualType(), Expr::Classification(), true, [](ArrayRef<QualType>, bool){ return false; }).

   f. If Result == TemplateDeductionResult::Success AND cand != nullptr, return cand.

3. Return nullptr if no match found.

IMPORTANT: This method runs inside the coroutine context so it MUST use the TaskQueue/InvocationContext->SwitchTo pattern for every clang operation. Wrap the entire logic in a TaskQueue.emplace lambda that receives TensorTypeHelpers &helper and CustomAstConsumer &, then SwitchTo(*CompileExecutionContext) after enqueueing. The Ctx and SemaRef are accessed via helper.Builder.Ctx and helper.Builder.SemaRef respectively.

Use the exact same coroutine pattern as the existing methods: enqueue the lambda then call InvocationContext->SwitchTo(*CompileExecutionContext).
