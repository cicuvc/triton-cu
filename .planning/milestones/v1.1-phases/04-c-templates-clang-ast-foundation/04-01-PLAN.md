  Per D-03/D-04: SharedTensor holds zero-length T data[] with operator() returning T&. Per D-05: test function uses SharedTensor& params. Per D-06: evaluate() fully implemented — blockIndices accepted but unused (Phase 4 passes `{}`).

  A. OffsetBases<RANK> NTTP carrier struct (D-01, D-02):
     - Template parameter: uint32_t RANK
     - Field: `IntTuple<RANK> Dims[]` with flexible-array-like size; actual array length set via class specialization
     - constexpr default constructor (zeros Dims)
     - constexpr constructor from `std::initializer_list<IntTuple<RANK>>`

  B. BlockBases<RANK> NTTP carrier struct:
     - Identical structure to OffsetBases<RANK>
     - Separate type for type-safety (so SharedLinearLayout template args are distinguishable)

   C. SharedLinearLayout<OffsetBases OB, BlockBases BB, uint32_t Alignment> template (D-01, D-02, D-06):
      - NTTP parameters: `OB` of some OffsetBases specialization, `BB` of some BlockBases specialization, `Alignment` as uint32_t
      - static constexpr uint32_t RANK = decltype(OB)::RANK (derived from the OffsetBases type)
      - static constexpr uint32_t Align = Alignment
      - No `REG_SIZE` member (SharedLinearLayout is NOT a register layout — it computes byte offsets into shared memory)
      - Does NOT accept a `Shape<...>` template argument and has NO access to tensor strides — it is Shape-independent.

      - **Core method: `static constexpr IntTuple<RANK> evaluate(uint32_t flatIndex, const IntTuple<RANK>& blockIndices)`:**
        * Takes a single flat `uint32_t` (the bit-space encoding of the logical index — computed by `SharedTensor::operator()` using TShape strides) and an `IntTuple<RANK>` blockIndices.
        * Mirrors the existing `BasisGroup::evaluate(uint32_t x)` pattern (tt_plugin.cu:45-49) EXACTLY: for each bit position `i`, if bit `i` of `flatIndex` is set, XOR-add `OB.Dims[i]` to the result via `IntTuple::operator+`.
        * Similarly for block bases: for each bit position `i`, if bit `i` of the block-index encoding is set, XOR-add `BB.Dims[i]`.
        * Combined result = offset_contrib + block_contrib (element-wise XOR via IntTuple::operator+).
        * `blockIndices` accepted but unused this phase (D-06: "Phase 4: blockIndices = {}"); the parameter is present so the interface is stable for future phases.
        * The return value is `IntTuple<RANK>` — a per-dim logical coordinate (bit-identical to MLIR `LinearLayout.apply()` output for the same offset/block bases at `gluon_ir.cc:461-463`).
        * SharedLinearLayout does NOT flatten logical coordinates to a flat index — that step lives in `SharedTensor::operator()` because only SharedTensor has `TShape` (stride) knowledge. evaluate() operates purely on the bit-space flat index.
        * **D-07 parity contract:** For the same offset/block bases and the same flatIndex input, `evaluate()` output MUST be bit-identical to MLIR `LinearLayout({offsetBases, blockBases}, outDims).apply()`.

      - **No REG_SIZE member.** SharedLinearLayout is NOT a register layout — it maps bit positions to logical-dim offsets, not register indices.

   D. SharedTensor<T, TShape, TLayout> template (D-03, D-04, D-05):
      - Template params: typename T (scalar type), typename TShape (Shape<DIMS...>), typename TLayout (SharedLinearLayout<...>)
      - `T data[]` — zero-length array at struct end (D-03: aliases external shared memory, no owned storage; lowers cleanly to ptr addrspace(3) in Phase 6)

      - **Core method: `__device__ T& operator()(auto... indices)` — variadic multi-index (D-04: "logical indices" plural):**
        * Accepts `RANK` arguments (one per dimension), each implicitly convertible to `uint32_t`.
          Implementation uses a fold-expression: construct `IntTuple<RANK>{static_cast<uint32_t>(indices)...}`.
          For Rank-1 `Shape<32>`: called as `shm(0)`, `shm(i)`. For Rank-2 `Shape<32,16>`: called as `shm(0, 1)`, `shm(i, j)`.
        * Flatten the logical indices to a single `uint32_t flatIndex` using `TShape`'s dims:
          Convention: row-major order — `DIMS...` are interpreted as [outer_dim, inner_dim] where inner_dim varies fastest.
          For `Shape<D0>` (Rank 1): `flatIndex = indices[0]`.
          For `Shape<D0, D1>` (Rank 2): `flatIndex = indices[0] * D1 + indices[1]`.
          For `Shape<D0, D1, D2>` (Rank N): `flatIndex = sum(indices[k] * stride[k])` where `stride[k] = product of DIMS[k+1..N-1]`.
          This flattening MUST be constexpr-compatible for device-side evaluation.
        * Call `TLayout::evaluate(flatIndex, IntTuple<RANK>{})` to get the per-dim logical offset `IntTuple<RANK>`.
        * Convert the `IntTuple<RANK>` logical offset to a 1D byte offset:
          `byteOffset = dot(logicalOffset, byteStrides)` where `byteStrides[k] = product(DIMS[k+1..N-1]) * sizeof(T)`.
          The result is the byte index into `data[]`.
        * Return `data[byteOffset]` as `T&` (D-04: mutable reference = read+write).
        * The return type `T&` is critical — the executor must verify the function signature literally returns `T&` (not `T` or `const T&`).

      - **Separation of concerns:** `SharedTensor` owns the logical→flat→byte-offset computation using `TShape` knowledge.
        `SharedLinearLayout::evaluate()` operates purely in bit-space (flat uint32_t → IntTuple<RANK>), remaining Shape-independent.
      - No separate `load()`/`store()` methods; no `operator[]`. All access goes through `operator()` (D-04).

   E. Test-only device functions (D-05 exercise):
      - After the template definitions, add two test device functions for Phase 4 verification.
        Both use `SharedTensor<...>&` parameters per D-05:

        (i) 1D write function (used by Plan 04-03 round-trip test):
        ```

        __device__ void write_shared_1d(SharedTensor<T, Shape<N>, TLayout>& shm, T val) {

        }
        ```
        Calls `operator()(0)` — single logical index for a 1D Shape<N>. Validates the variadic operator() compiles with arity=1.

        (ii) 2D read+write function (exercises multi-index path of operator()):

        ```
            for (int i = 0; i < 32; i++)
                for (int j = 0; j < 16; j++)
                    shm(i, j) = shm(i, j) * scale;
        }
        ```
        Calls `operator()(i, j)` — two logical indices for a 2D Shape<32,16>. Validates the variadic operator() compiles with arity=2 and returns T& (both read and write).

      - Uses `SharedTensor<...>&` parameter (D-05) — mutable reference.
      - Exercises operator() read (RHS of assignment) and operator() write (LHS of assignment).
      - These functions are consumed by Plan 04-03's pytest harness (GPU-free, compiled via SuspendedCudaCompiler but never launched).
